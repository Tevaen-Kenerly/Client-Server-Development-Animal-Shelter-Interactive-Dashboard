# Client-Server-Development-Animal-Shelter-Interactive-Dashboard
Interactive Python dashboard for Grazioso Salvare using MongoDB, CRUD module, and data visualizations to support animal rescue decisions.

# How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future? 
I write programs to be maintainable, readable, and adaptable by separating responsibilities, using clear naming, and keeping logic modular. In this course, the best example was the CRUD Python module from Project One—by isolating all database operations in one place, I could plug that module directly into the Project Two dashboard without rewriting queries. The advantage of working this way is that if the database changes (new collection, new fields, or even a different database engine), I only need to update the CRUD layer instead of touching every widget. In the future, I could reuse this CRUD module pattern for other dashboards, APIs, or internal tools that need to talk to MongoDB in a consistent way.

When I approach a problem as a computer scientist, I start by understanding the requirements and constraints before I write any code.
